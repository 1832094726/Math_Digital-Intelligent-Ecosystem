# 数字智能数学教育生态系统（DIEM）

## 项目概述

数字智能数学教育生态系统（DIEM）是一个综合性教育平台，旨在通过缓解自闭和统一实现连通性、定制化和智能化，为K-12阶段的数学教育提供全方位、个性化的学习支持。本系统整合了知识图谱、智能推荐、自适应学习等先进技术，构建了一个连接学生、教师、家长的教育生态环境，实现数学学习过程的智能化、个性化与高效化。

## 系统目标

- 实现连通性：打破学习场景之间的壁垒，实现无缝连接
- 提供定制化：针对不同学习场景提供专属解决方案
- 增强智能化：通过AI技术为教与学提供智能服务支持
- 构建数字化教育平台，提升K-12数学教育质量和效率
- 为不同学习阶段的学生提供个性化学习路径
- 为教师提供智能教学工具和数据分析能力
- 促进家校协同，形成教育合力

## 系统架构

本系统采用三层架构设计：

1. **云基础设施层**：用于资源同步、用户管理和安全的中心协调枢纽
2. **场景应用层**：为八个场景提供的自定义接口，每个接口都符合标准API规范以实现互操作性
3. **AI服务层**：提供高级问题解决、符号推荐和自动评分的后端智能模块

## 八大核心场景

DIEM系统围绕以下八个核心场景构建，通过教育云平台实现互联互通：

### 1. 课堂授课
课堂教学环境中教师进行数学知识讲解、演示和互动的场景。

### 2. 自学
学生独立进行知识学习、复习和练习的场景。

### 3. 备课
教师准备课程内容、设计教学活动和评估材料的场景。

### 4. 家庭学习
家庭环境中，学生在家长陪伴下进行学习的场景。

### 5. 同伴学习
学生之间互相协作、讨论和共同解决问题的场景。

### 6. 做作业
学生完成老师布置的作业任务的场景。本系统特别强化了作业管理互动方案，包括作业接收、完成和提交的完整工作流。

### 7. 小组学习
学生组成学习小组进行协作学习的场景。

### 8. 小组作业
学生团队共同完成项目型作业的场景。

## 交互方案示例

### 学生作业管理交互方案

作业是学生的高频活动，系统提供完整的作业管理交互方案：

**动作1 (作业接收)**：
- 学生登录数字智能作业管理系统(DIM)接收作业
- 系统与学生互动，为每项作业建立个性化提醒计划

**动作2 (作业完成)**：
- 设备选择：学生选择合适的设备完成作业
- 问题解决：系统提供符号推荐支持，包括基于常见错误的建议
- 解答复查：学生可请求评分工具评估解答
- 解答修改：学生根据评分工具反馈修改作业

**动作3 (作业提交)**：
- 学生登录DIM系统提交完成的作业

### 教师作业管理交互方案

教师侧作业管理主要包含以下动作：

**动作1 (作业分发)**：
- 教师登录数字智能作业管理系统(DIM)向学生分发作业
- 系统与教师协作，为每项作业建立定制提醒计划

**动作2 (作业评分)**：
- 情况1：教师利用自动评分工具评估学生提交的作业
- 情况2：教师手动审核预评分的提交内容进行质量保证

**动作3 (补救支持)**：
- 系统通过作业评估表和错误模式追踪进行全面分析
- 识别特定弱点(理解差距、概念应用错误或计算错误)以指导有针对性的干预

**动作4 (作业反馈)**：
- AI驱动工具执行作业提交的比较分析
- 向学生提供个性化反馈
- 将所有数据归档于系统以供未来参考

## 主要组件

目前系统包含两个主要组件：

### 1. Subject_symbol_dynamic_keyboard（学科符号动态键盘）

数学学科符号动态键盘是一个智能的数学符号输入工具，能够根据用户的输入内容和上下文，动态预测并推荐可能需要的数学符号，极大地提升了数学内容的输入效率。

- **board-backend**：基于Python和Flask实现的后端服务，提供符号预测API
- **board-frontend**：基于Vue.js实现的前端界面，提供符号键盘UI和交互功能

#### 功能特点：
- 上下文感知的符号推荐
- 多级符号分类展示
- 支持LaTeX格式输出
- 知识图谱驱动的智能推荐

### 2. learning-front（学习前端）

学习前端是系统的统一用户界面，整合了各种学习服务，为学生提供个性化的学习体验。

- 基于Vue.js构建的前端应用
- 模块化设计，支持各类学习功能接入
- 自适应界面，支持多终端访问

#### 功能特点：
- 个性化学习控制面板
- 自适应练习与评测
- 学习路径可视化
- 智能答疑交互
- 学习进度追踪

## AI服务

### 1. 数学解题引擎
旨在自动化解决数学问题及评分学生的步骤解题过程。当缺少解题路径时，系统会生成解题路径以辅助自动评分。

### 2. 作业自动评分
能够分析学生提交的作业，提供智能评分和反馈，并识别常见错误模式以进行针对性指导。

### 3. 符号推荐系统
基于上下文和知识点分析，动态预测并推荐用户可能需要的数学符号，提高数学表达效率。

## 系统集成与数据流

本生态系统通过以下方式实现组件间的集成与数据流转：

1. **知识集成**：基于数学知识图谱，将教学内容、练习题目与知识点关联
2. **学习分析**：收集学习行为数据，通过分析形成学习者画像
3. **个性化服务**：基于知识图谱和学习者画像，提供个性化学习路径和内容推荐
4. **反馈优化**：学习结果数据反馈至系统，不断优化推荐算法和内容展示

## 技术实现

- **前端技术**：Vue.js, Vuex, Vue Router, ElementUI
- **后端技术**：Python, Flask, RESTful API
- **数据存储**：关系型数据库与NoSQL数据库混合架构
- **算法引擎**：知识图谱算法、推荐算法、自然语言处理
- **部署架构**：基于Docker的云原生架构
