用户认证流程

0. 用户登录与身份验证

输入用户名和密码，系统验证身份

区分教师和学生身份，授予相应权限

认证模块

接口: POST /api/auth/login

验证用户凭据，生成JWT令牌

接口: GET /api/auth/me

获取当前登录用户信息

缓存层

操作: SET user:{id} {user_info} EX 3600

缓存用户基本信息，减少数据库查询

数据库

操作: SELECT * FROM users WHERE username=? AND password=?

验证用户信息并获取用户角色

前端设计

组件: QuestionCreationSelector

组件: ResourceDropdown 资源下拉框

本地缓存: 资源列表

下拉选择器、参数配置界面

资源树形结构展示

数据库表结构

users 表

  • id: INT (主键)
  • username: VARCHAR (用户名)
  • password: VARCHAR (加密密码)
  • role: ENUM ('teacher', 'student')
  • grade: INT (年级，学生特有)
  • created_at: DATETIME
  • updated_at: DATETIME

resources 表

  • id: INT (主键)
  • name: VARCHAR (资源名称)
  • type: ENUM ('knowledge', 'material', 'video')
  • description: TEXT (描述)
  • subject: VARCHAR (所属学科)
  • grade: INT (适用年级)
  • parent_id: INT (父资源ID，用于层级结构)

questions 表

  • id: INT (主键)
  • teacher_id: INT (外键关联users)
  • resource_id: INT (外键关联resources)
  • content: TEXT (题目内容)
  • score: INT (分值)
  • answer: TEXT (参考答案)
  • difficulty: ENUM ('easy', 'medium', 'hard')
  • status: ENUM ('draft', 'published')
  • visible_to: JSON (可见学生范围)
  • due_time: DATETIME (截止时间)
  • created_at: DATETIME
  • updated_at: DATETIME

answers 表

  • id: INT (主键)
  • question_id: INT (外键关联questions)
  • student_id: INT (外键关联users)
  • content: TEXT (学生答案)
  • submitted_at: DATETIME
  • status: ENUM ('submitted', 'graded')

gradings 表

  • id: INT (主键)
  • answer_id: INT (外键关联answers)
  • teacher_id: INT (外键关联users)
  • score: INT (得分)
  • comment: TEXT (教师评语)
  • graded_at: DATETIME

notifications 表

  • id: INT (主键)
  • user_id: INT (接收用户ID)
  • type: ENUM ('new_question', 'new_answer', 'graded')
  • content: TEXT (通知内容)
  • related_id: INT (关联ID，如题目ID)
  • is_read: BOOLEAN (是否已读)
  • created_at: DATETIME

模块间交互关系

模块 依赖模块 主要交互方式 交互目的
认证模块 - 提供API接口 用户身份验证、权限控制
资源模块 认证模块 提供API接口 提供教学资源数据支持
出题模块 认证模块、资源模块、通知模块 调用API、事件触发 获取教学资源、触发通知
作答模块 认证模块、出题模块、通知模块 调用API、事件触发 验证题目、触发通知
批改模块 认证模块、出题模块、作答模块、通知模块、资源模块 调用API、事件触发 获取作答、获取相关资源、触发通知
通知模块 认证模块 提供API接口、事件监听 处理各模块的通知需求
